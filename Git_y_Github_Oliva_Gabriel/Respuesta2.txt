¿Quién inventó el sistema de control de versión Git y por qué?

Git es un software de código abierto de control de versiones diseñado y 
desarrollado originalmente por Linus Torvalds, el famoso creador del kernel 
del sistema operativo Linux, en 2005.
Pensando en la eficiencia, la confiabilidad y compatibilidad del 
mantenimiento de versiones de aplicaciones cuando 
estas tienen un gran número de archivos de código fuente
para controlar el desarrollo del kernel de Linux.

¿A quién pertenece actualmente Github y por qué?

Actualmente pertenece a Microsoft a partir de Junio de 2018.
La compañía adopta una estrategia más abierta al 
mundo del software libre y a la integración con otras plataformas y servicios. 
Un ejemplo es la adquisición en 2016 de Xamarin, una plataforma que permite 
crear aplicaciones móviles que pueden funcionar en distintos SO.
Además, hay otro motivo de peso: Microsoft Azure. En plena explosión de cloud 
services (servicios en la nube), con un crecimiento a la vista del año del 
21.4%, Microsoft tiene que competir con el líder,la todopoderosa Amazon con AWS. 
Qué mejor forma de hacerlo que estar en contacto directo con los desarrolladores 
que trabajan en esos servicios, ver sus necesidades y ofrecerles herramientas 
que faciliten su integración con Azure.
